In HuggingGPT, task planning is the first but also
the most important step in the whole workflow,
since it directly determines the result of the sub-
sequent pipeline. Therefore, we deem that the
quality of task planning also reflects the capabil-
ity of LLMs in some aspects (e.g., reasoning and
decision). In other words, assessing task planning
provides a reliable metric for evaluating the ability
of LLMs. We simplified the evaluation by only
considering the task type without its arguments. To better conduct evaluations on task planning, we
group tasks into three distinct categories (see Table 2) and formulate different metrics on them:

\begin{itemize}
\item \textbf{Single Task} refers to a request that involves only one task. We consider the planning to be correct
when and only when the task name (i.e., "task") and the predicted label are identically equal. F1
and accuracy are chosen as evaluation metrics for single-task scenarios.
\item \textbf{Sequential Task} represents that the intent of the user request can be decomposed as a sequence of
multiple sub-tasks. Here, we use F1 and normalized Edit Distance [35] as the metrics.
\item \textbf{Graph Task} indicates that user requests can be decomposed into directed acyclic graphs. Con-
sidering the possibility of multiple planning topologies within graph tasks, only using F1-score is
not enough to reflect the LLM capability in planning. To address this, following Vicuna [36], we
employed GPT-4 as a critic to evaluate whether the planning is correct. The accuracy is obtained
by evaluating the judgment of GPT-4, referred to as the GPT-4 Score. Detailed information about
the GPT-4 Score can be found in Appendix A.1.5.
\end{itemize}

\paragraph{Dataset}
To conduct our evaluation, we invite
some annotators to submit some requests. We
collect these data as the evaluation dataset. We
use GPT-4 to generate task planning as the pseudo
labels, which cover single, sequential, and graph
tasks. Furthermore, we also invite some expert
annotators to label task planning for some complex
requests (46 examples) as a high-quality human-
annotated dataset. We also plan to further improve
the quality and quantity of this dataset to better
help us to evaluate the LLM capability in planning, which leaves as future work. More details about
this dataset are in Appendix A.2. Based on this dataset, we deploy some experimental evaluations on
different LLMs (Alpaca-7b [37], Vicuna-7b [36], and GPT models) for task planning.
