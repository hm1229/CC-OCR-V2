\texttt{+--------------------------------------------------------------------------------+}
\end{center}

\noindent This function was added in MySQL 8.0.3 as a synonym for \texttt{JSON\_MERGE()}. The \texttt{JSON\_MERGE()} function is now deprecated, and is subject to removal in a future release of MySQL.

\noindent This function is similar to but differs from \texttt{JSON\_MERGE\_PATCH()} in significant respects; see \textcolor{blue}{JSON\_MERGE\_PATCH() compared with JSON\_MERGE\_PRESERVE()}, for more information.

\begin{itemize}

\item \texttt{\textcolor{blue}{JSON\_REMOVE(}}\textit{\textcolor{blue}{json\_doc, path[, path] ...}}\texttt{\textcolor{blue}{)}}

Removes data from a JSON document and returns the result. Returns \texttt{NULL} if any argument is \texttt{NULL}. An error occurs if the \textit{json\_doc} argument is not a valid JSON document or any \textit{path} argument is not a valid path expression or is \texttt{\$} or contains a \texttt{*} or \texttt{**} wildcard.

The \textit{path} arguments are evaluated left to right. The document produced by evaluating one path becomes the new value against which the next path is evaluated.

It is not an error if the element to be removed does not exist in the document; in that case, the path does not affect the document.

\begin{lstlisting}
mysql> SET @j = '["a", ["b", "c"], "d"]';
mysql> SELECT JSON_REMOVE(@j, '$[1]');
+------------------------+
| JSON_REMOVE(@j, '$[1]') |
+------------------------+
| ["a", "d"]             |
+------------------------+
\end{lstlisting}

\item \texttt{\textcolor{blue}{JSON\_REPLACE(}}\textit{\textcolor{blue}{json\_doc, path, val[, path, val] ...}}\texttt{\textcolor{blue}{)}}

Replaces existing values in a JSON document and returns the result. Returns \texttt{NULL} if any argument is \texttt{NULL}. An error occurs if the \textit{json\_doc} argument is not a valid JSON document or any \textit{path} argument is not a valid path expression or contains a \texttt{*} or \texttt{**} wildcard.

The path-value pairs are evaluated left to right. The document produced by evaluating one pair becomes the new value against which the next pair is evaluated.

A path-value pair for an existing path in the document overwrites the existing document value with the new value. A path-value pair for a nonexisting path in the document is ignored and has no effect.

In MySQL 8.0.4, the optimizer can perform a partial, in-place update of a \texttt{JSON} column instead of removing the old document and writing the new document in its entirety to the column. This optimization can be performed for an update statement that uses the \texttt{JSON\_REPLACE()} function and meets the conditions outlined in \textcolor{blue}{Partial Updates of JSON Values}.

For a comparison of \texttt{JSON\_INSERT()}, \texttt{JSON\_REPLACE()}, and \texttt{JSON\_SET()}, see the discussion of \textcolor{blue}{\texttt{JSON\_SET()}}.

\begin{lstlisting}
mysql> SET @j = '{ "a": 1, "b": [2, 3]}';
mysql> SELECT JSON_REPLACE(@j, '$.a', 10, '$.c', '[true, false]');
+-----------------------------------------------------+
| JSON_REPLACE(@j, '$.a', 10, '$.c', '[true, false]') |
+-----------------------------------------------------+
| {"a": 10, "b": [2, 3]}                              |
+-----------------------------------------------------+
\end{lstlisting}
